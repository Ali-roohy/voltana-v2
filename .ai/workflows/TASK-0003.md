## API Contract (architect — 2026-05-30)

**All `/v1/*` routes require a valid Bearer access token** (the `middleware.Auth` already
wired sets `UserIDKey`). Missing/invalid token → `401` (handled by middleware).

**User identity:** handlers read `userID := c.MustGet(middleware.UserIDKey).(uuid.UUID)` and
pass it into the service → repository. **The repository MUST scope every `cars` query by
`user_id`.** Cross-user access returns **404 NOT_FOUND** (not 403) so existence is not leaked.

**Error envelope** (consistent with TASK-0009):
`{ "error": "<human message>", "code": "<MACHINE_CODE>" }`
Codes used here: `INVALID_REQUEST` (malformed/validation), `NOT_FOUND`, `INVALID_EV_MODEL`.

**List envelope** (for collection endpoints):
```json
{ "items": [ ... ], "limit": 20, "offset": 0, "total": 42 }
```
**Pagination query params:** `limit` (default `20`, min `1`, max `100`), `offset`
(default `0`, min `0`). Out-of-range → clamp (do not error).

### Car resource (response shape)
```json
{
  "id": "uuid",
  "ev_model_id": "uuid | null",
  "name": "Daily",
  "license_plate": "12ج345-67 | null",
  "odometer_km": 18250,
  "created_at": "RFC3339",
  "updated_at": "RFC3339"
}
```

### POST /v1/cars  → 201
Request:
```json
{ "name": "Daily", "ev_model_id": "uuid|null", "license_plate": "…|null", "odometer_km": 0 }
```
| Field | Rule |
|-------|------|
| `name` | required, 1–255 chars |
| `ev_model_id` | optional; if present must be a valid UUID **and** exist in `ev_models` → else `422 INVALID_EV_MODEL` |
| `license_plate` | optional, ≤ 50 chars |
| `odometer_km` | optional int, ≥ 0 (default 0) |

Responses: `201` car body · `400 INVALID_REQUEST` (bad JSON/validation) ·
`422 INVALID_EV_MODEL` · `401`.

### GET /v1/cars  → 200
Returns the **current user's** cars only, newest first (`created_at DESC`), paginated via the
list envelope. `total` = count of the user's cars.

### GET /v1/cars/:id  → 200 | 404
`200` car body if owned by the caller; `404 NOT_FOUND` otherwise (including not-owned).
`400 INVALID_REQUEST` if `:id` is not a UUID.

### PUT /v1/cars/:id  → 200 | 404
Full update (same body + rules as POST). Only the owner's row is updated
(`WHERE id=$1 AND user_id=$2`). `404 NOT_FOUND` if not owned. `422 INVALID_EV_MODEL` as above.

### DELETE /v1/cars/:id  → 204 | 404
Deletes the owner's row; `204 No Content` on success, `404 NOT_FOUND` if not owned.

### EV model resource (response shape)
```json
{
  "id": "uuid", "name_fa": "…", "name_en": "…", "brand": "…|null",
  "battery_capacity_kwh": 60.0, "range_km": 450, "chemistry": "LFP|NMC|NCA|null",
  "created_at": "RFC3339"
}
```

### GET /v1/ev-models  → 200
Query params: `q` (optional search term), `limit`, `offset` (as above).
- When `q` is empty → list all models, `name_en ASC`, paginated.
- When `q` is present → match on **either** name using the existing FTS gin indexes:
  `WHERE to_tsvector('simple', name_fa) @@ plainto_tsquery('simple', $q)
     OR to_tsvector('simple', name_en) @@ plainto_tsquery('simple', $q)`.
  (Whole-token match. If typeahead/substring is needed later, add a `pg_trgm` index —
  out of scope here.) Returns the list envelope; `total` = count of matches.

### GET /v1/ev-models/:id  → 200 | 404
`200` model body, `404 NOT_FOUND` if no such id, `400 INVALID_REQUEST` if `:id` not a UUID.
No `user_id` scoping (catalog is shared, read-only).
