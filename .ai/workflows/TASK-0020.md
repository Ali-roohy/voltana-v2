# TASK-0020 — Font Selection in Settings

**Status**: DONE ✅ CLOSED with caveat (qa_supervisor 2026-06-04)
**Phase**: 3 (UX feature)
**Assigned to**: feature → developer
**Reviewer**: dev_supervisor
**Persona**: pm (this spec) → feature (font list + loading strategy) → developer (implements) → dev_supervisor
**Depends on**: TASK-0006 (✅ frontend baseline)

## Goal
Let users choose their preferred typeface from a curated list. Since the app is primarily Persian, **Vazirmatn**
(current default) stays as-is; the selector adds alternatives for users who prefer a different Persian
rendering or Latin-primary workflow.

---

## Scope In
- **Font list** (5–6 options):
  - *Vazirmatn* — current default, stays selected on first load (no behavior change if unset)
  - *Dana* — alternative Persian sans-serif
  - *Samim* — Persian hand-friendly
  - *Yekan Bakh* — popular Persian modern (check license)
  - *Inter* — clean Latin/UI font (useful for Latin-heavy sessions/data views)
  - *IRANSans* — widely used Persian UI (check license — prefer libre/self-hosted)
- **Loading strategy** (feature persona to decide, flagged for developer):
  - **Self-hosted** (preferred): download WOFF2 files into `voltana-web/public/fonts/`, declare `@font-face`
    in `index.css`. No external request, works offline/on VPS with no CDN.
  - **Google Fonts** fallback only if a font has no libre license and can't be self-hosted.
  - All fonts must include Arabic/Persian subset (`U+0600–U+06FF`) or the selection is meaningless.
- **Font selector card** in `pages/Settings.tsx`: list of named options, current selection highlighted.
- **Persistence in localStorage** under key `voltana:font` (same rationale as TASK-0019 themes).
- Apply the chosen font by setting `document.documentElement.style.fontFamily` (or a CSS variable
  `--font-sans` overriding the Tailwind default) on mount and on change.
- **`lib/fonts.ts`** (or co-located with the theme lib from TASK-0019): font registry with name,
  CSS family stack, and whether it's self-hosted or CDN.

## Scope Out
- **Font size** adjustment (separate task if desired).
- Custom font upload.
- Backend persistence.
- Variable font axes (weight/width sliders).

## Key Design Decisions (for feature persona to resolve)
1. **Self-hosted vs CDN for each font**: feature persona to verify licenses and availability as
   libre WOFF2. Google Fonts CDN is blocked in Iran — **all fonts must work self-hosted**.
2. **Font file size budget**: WOFF2 with Persian subset only (~30–80 KB per weight). Do not load full
   Unicode range. One weight (Regular 400) is enough for the selector; Bold 700 optional.
3. **Fallback stack**: every `font-family` declaration should fall back to `Vazirmatn, sans-serif`
   so a missing font doesn't break Persian rendering.
4. **Integration with TASK-0019**: if both theme and font tasks land in the same PR, they can share
   a single `lib/preferences.ts` module. If separate, keep them independent.

## Acceptance Criteria
- [ ] 5–6 fonts available in the Settings selector, all with Persian subset.
- [ ] Selecting a font changes the UI typeface immediately across all pages.
- [ ] Preference persists on page reload (localStorage).
- [ ] Default is Vazirmatn — selecting nothing changes the current look.
- [ ] All fonts load **without a CDN request** (self-hosted WOFF2 in `public/fonts/`).
- [ ] `npx tsc --noEmit` and `npm run build` pass; no new network errors in browser console.

## Handoff → feature
Verify font licenses, download WOFF2 Persian-subset files, produce the `@font-face` declarations, the
font registry for `lib/fonts.ts`, and the Settings card layout spec. Flag any font that can't be
self-hosted so pm can cut it. Hand off to developer with the registry + `@font-face` CSS ready.

production bug, not a feature. Feature persona owns license research and WOFF2 prep. Reviewer:
