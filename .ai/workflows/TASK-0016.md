## Goal
Give admins a self-service UI to manage charging stations. TASK-0013 shipped the `/v1/stations` API
(admin-gated writes) and the user-facing read-only map, but **station create/edit/delete is currently
out-of-band SQL only**. This task adds an **admin-only `/admin/stations` page** that drives the existing
API end to end ‚Äî list, add, edit, delete ‚Äî with a Leaflet map preview for placing coordinates.

No backend, DB, or migration work: the `/v1/stations` endpoints and the `is_admin` boundary already exist.

---

## Scope In
- Admin-only page at **`/admin/stations`**.
- **List** all stations in a table view (name, operator, address, lat/lng, connector types, max power).
- **Add** new station form: `name`, `operator`, `address`, `latitude`, `longitude`, `connector_types`,
  `max_power_kw`.
- **Edit** an existing station (same fields, pre-filled).
- **Delete** a station, behind a **confirmation** step.
- **Map preview** on the add/edit form (Leaflet, reusing the same setup as `Map.tsx`) ‚Äî clicking/dragging
  the marker updates the lat/lng fields and vice-versa.
- Page (and any nav entry to it) **only visible to `is_admin=true` users**.

## Scope Out
- The user-facing read-only station map (already shipped in TASK-0013).
- Bulk import / CSV upload of stations.
- Station photos / image upload.
- Any new backend endpoint, migration, or change to the `AdminOnly` boundary.

## Acceptance Criteria
- [ ] **Non-admin users cannot access `/admin/stations`** ‚Äî route guard redirects to `/` (and the nav entry
      is hidden for non-admins). Admin status comes from the authenticated user (`is_admin`).
- [ ] **All CRUD operations work via the existing `/v1/stations` API** ‚Äî list (GET), create (POST),
      update (PUT), delete (DELETE) ‚Äî through `features/stations/{api.ts,hooks.ts}` (no direct `fetch()` in
      components, per ADR-002), with TanStack Query cache invalidation on each mutation.
- [ ] **Form validation:** required fields enforced; `latitude` in **[-90, 90]**, `longitude` in
      **[-180, 180]**; `max_power_kw` positive. Surface server-side 400/403 errors to the user (e.g. a
      non-admin or revoked-admin hitting a write gets a clear message, not a silent failure).
- [ ] **Reuses existing UI components** where possible (e.g. the shared `src/components/` primitives and the
      `TOUBreakdown`/card patterns), and the same Leaflet/react-leaflet (**v4**) setup as `Map.tsx` ‚Äî no new
      map dependency, no react-leaflet bump.
- [ ] `cd voltana-web && npx tsc --noEmit && npm run build` both pass.

## Notes / pointers (for feature + developer)
- Frontend data layer already exists from TASK-0013: `voltana-web/src/features/stations/api.ts` +
  `hooks.ts`. Extend these with create/update/delete hooks rather than adding a parallel layer.
- The map component and marker logic live alongside `Map.tsx` (TASK-0013) ‚Äî factor out a small reusable
  map-with-marker piece if it eases the add/edit preview; keep shared UI in `src/components/`.
- Admin gating: a route guard reading the current user's `is_admin`. Remember `AdminOnly` does a **fresh DB
  check** server-side, so the UI guard is convenience only ‚Äî the API remains the real boundary (a
  revoked-admin must still get a clean 403 in the UI).
- Currency/units: `connector_types` and `max_power_kw` shapes follow the `charging_stations` schema from
  migration `000007`. Confirm exact field names against `features/stations/api.ts` types.

# Implementation Log ‚Äî developer (2026-06-03)

## ‚öÝÔ∏è Scope change (operator-approved): added backend `GET /v1/me`
The spec said "no backend work" **and** "route guard reading the user's `is_admin`" ‚Äî but the backend
exposed **no admin signal to the client**: `is_admin` is deliberately kept OUT of the access token
(TASK-0013, for instant revocation) and there was **no `/me` / user-info endpoint**. The guard was
therefore unimplementable as written. Operator chose **"Add GET /v1/me"** (over a frontend-only 403-reliance
or a hacky admin-endpoint probe). This grows the task to touch the backend + adds **security** as co-reviewer.

### Backend
- **`GET /v1/me`** (under the `/v1` Auth group) ‚Üí `{ id, email, is_admin }` for the authenticated user.
  - `internal/service/auth_service.go` ‚Äî new `GetUser(ctx, userID)` (wraps `users.FindByID`).
  - `internal/handler/auth_handler.go` ‚Äî new `Me` handler (reads `middleware.UserIDKey`); imports `middleware` + `uuid`.
  - `cmd/server/main.go` ‚Äî `v1.GET("/me", authH.Me)`.
  - `internal/service/auth_service_test.go` ‚Äî `TestGetUser_ReturnsIdentityWithAdminFlag` + `TestGetUser_UnknownID`.
  - **`AdminOnly` boundary unchanged** ‚Äî the API remains the real authz boundary; `/v1/me` only *reports* the flag.

### Frontend
- **`features/auth/api.ts`** ‚Äî `Me` type + `getMe()`. **`features/auth/hooks.ts`** (new) ‚Äî `useMe(enabled)`
  (`retry:false` so a 401/403 surfaces to the guard; `staleTime` 5m).
- **`features/stations/api.ts`** ‚Äî `StationInput` + `createStation`/`updateStation`/`deleteStation` (shared
  `toPayload` omits blank optionals so the server's `omitempty` validators pass). **`features/stations/hooks.ts`**
  ‚Äî `useCreateStation`/`useUpdateStation`/`useDeleteStation`, each invalidating `["stations"]` (ADR-002: no
  `fetch()` in components).
- **`components/StationMapPicker.tsx`** (new, shared) ‚Äî Leaflet/OSM picker reusing the `Map.tsx` marker-icon
  fix; click-to-place + draggable marker + recenter-on-type, coords rounded to 6 dp. **react-leaflet v4 kept.**
- **`components/AdminRoute.tsx`** (new) ‚Äî guard: not-logged-in ‚Üí `/auth`; logged-in non-admin / `me` error ‚Üí `/`;
  spinner while auth + `me` settle.
- **`pages/AdminStations.tsx`** (new) ‚Äî table list, add/edit dialog (shadcn `Dialog`) with the map picker +
  client validation (required name; lat ‚àà [-90,90]; lng ‚àà [-180,180]; power positive int), delete behind a
  shadcn `AlertDialog` confirm. **API errors surfaced** (403 ‚Üí "admin only" toast). Reuses existing shadcn UI.
- **`App.tsx`** ‚Äî `/admin/stations` route wrapped in `<AdminRoute>`. **`Header.tsx`** ‚Äî admin-only Shield nav
  button (hidden unless `me.is_admin`).

## Deviations / decisions
1. **Field name `power_kw`, not `max_power_kw`** ‚Äî the spec said `max_power_kw`, but the TASK-0013 API +
   `StationMarker` use **`power_kw`** (int, `min=1`). Followed the actual API (spec note ¬ß60 says confirm
   against `api.ts`). No `max_power_kw` exists server-side.
2. **Table operator/address via per-row detail hydration** ‚Äî `GET /v1/stations` returns *markers only* (no
   operator/address). To honour the spec's columns, the page hydrates each row via `getStation(id)` (TanStack
   `useQueries`, cached under `["stations", id]`, reused on Edit). Bounded reference data, so the N+1 is
   acceptable; Edit is disabled until a row's detail has loaded. (Alternative ‚Äî widen the list payload ‚Äî would
   be more backend scope; deferred.)
