## Problem

The TASK-0017 bot poller logs the full getUpdates URL **including the bot token** on network
errors (Go's `*url.Error` stringifies the request URL). Seen live in `docker logs voltana-api`:

```
bot: bale getUpdates error (retry in 5s): Get "https://tapi.bale.ai/bot1316617363:7YHB…/getUpdates?…": read tcp …: connection reset by peer
```

Anyone with log access (or a production log aggregator) can steal both bot tokens.

## Audit (pm pre-pass, 2026-06-12) — token-bearing URLs that can reach logs

| Site | Path to logs | Verdict |
|---|---|---|
| `Poller.getUpdates` — raw `client.Do` err | `Run()` `log.Printf(… %v)` (the live leak) | **FIX** |
| `Poller.sendJSON` — raw `NewRequest`/`Do` err | bubbles into `handleUpdate` log lines | **FIX** |
| `ProbeBot` — raw `client.Get` err | `main.go` "Telegram unreachable (%v)" | **FIX** |
| `sender.go sendMessage` — `fmt.Errorf(… %w, err)` | auth service "otp: … send failed: %v" | **FIX** |
| `bot/check.go` ConnectionTester | already sanitized (BUG-8) | OK |
| `scripts/healthcheck.sh` | `curl -s … > /dev/null \|\| true` — URL never echoed; `$MSG`/`$LOG` contain only `HEALTH_URL` | OK (audited, no change) |

## Fix

- Package-level `maskBotToken(s)` (regex `/bot[^/\s"]+` → `/bot***`) + `sanitizeURLErr(err)`
  in `internal/bot` — token-agnostic, masks any platform's token in any error text.
- Apply at every raw-HTTP-error return in `poller.go` (`getUpdates`, `sendJSON`, `ProbeBot`)
  and `sender.go` (`sendMessage`).
- Regression tests: force real HTTP errors against `http://127.0.0.1:1/bot<FAKE_SECRET>` and
  assert the returned error text contains no token substring; unit test for the mask itself.

## Acceptance Criteria

1. A forced poller network error logs `…/bot***/getUpdates…` — zero token characters.
2. `sendMessage` / `ProbeBot` errors equally masked.
3. Regression tests in `internal/bot` pass; full `go test ./...` green.
4. qa: live `docker logs voltana-api` grep after a forced error → zero token occurrences.

## Scope Out

- Rotating the (already-exposed) tokens — operator action, recommended post-merge.
- healthcheck.sh changes (audited clean).

## Implementation (developer, 2026-06-12)

- New `internal/bot/sanitize.go`: `maskBotToken` (regex `/bot[^/\s"]+` → `/bot***`, token- and
  platform-agnostic) + `sanitizeURLErr` (new error from masked text — deliberately NOT wrapped,
  so the token can't resurface via `errors.Unwrap`/`%+v`).
- Applied at every raw-HTTP-error return: `Poller.getUpdates` (×2), `Poller.sendJSON` (×2),
  `ProbeBot`, `sender.sendMessage` (×2 — incl. the `NewRequest` error, which also embeds the URL;
  the previous `%w` wrap replaced with `%v` of the sanitized error).
- `bot/check.go` (BUG-8) already safe; `scripts/healthcheck.sh` audited clean (URL never echoed).
- Regression tests `internal/bot/sanitize_test.go` (5): real HTTP failures against
  `http://127.0.0.1:1/bot<FAKE>` for getUpdates / sendJSON / ProbeBot / sendMessage — assert no
  token AND the `/bot***` marker present — plus a mask unit test on the exact live leak format.
