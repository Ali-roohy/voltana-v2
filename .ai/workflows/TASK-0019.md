# TASK-0019 — Theme System with Car Colors

**Status**: DONE ✅ CLOSED (qa_supervisor 2026-06-04)
**Phase**: 3 (UX feature)
**Assigned to**: feature → developer
**Reviewer**: dev_supervisor
**Persona**: pm (this spec) → feature (design theme palette + component spec) → developer (implements) → dev_supervisor
**Depends on**: TASK-0006 (✅ frontend baseline)

## Goal
Give Voltana a set of **artist-designed color themes** inspired by popular EV models and car colors. Users
pick a theme in Settings and the whole UI updates instantly via CSS custom properties. The default theme
keeps the current dark-electric look; presets are additive and never break the baseline.

---

## Scope In
- **6–8 preset themes**, each defined as a set of CSS custom-property overrides. Suggested names:
  - *Voltana Dark* (current default — keep unchanged)
  - *Tesla Red* (deep crimson accent, dark base)
  - *Leaf Green* (electric green, dark forest base)
  - *Ocean Blue* (deep-sea blue, navy base)
  - *Midnight Black* (near-black, high-contrast white accents)
  - *Arctic White* (light/white base — light-mode variant)
  - *Sunset Orange* (amber-orange accent)
  - *Sakura Pink* (soft pink accent, light-ish base)
- **Theme selector card** in `pages/Settings.tsx`: grid of swatches (primary + background color chip),
  active theme highlighted with a ring.
- **Instant preview**: selecting a theme applies immediately (no save button needed for the visual effect;
  the persistence write can be debounced).
- **Persistence in localStorage** under key `voltana:theme` (no backend change, no migration).
  Rationale: themes are a device preference; syncing across devices is a Phase-4 concern. A future
  backend sync is possible by adding a `theme` column to `user_settings` (flag to pm when scoped).
- **`ThemeProvider`** React context (or a `lib/theme.ts` helper) that reads localStorage on mount and
  applies the theme to `document.documentElement` via CSS variable overrides.
- All theme tokens are **CSS custom properties** on `:root` — no Tailwind config changes, no hard-coded
  hex values in components. Existing components pick up themes automatically.

## Scope Out
- Custom user-defined themes (hex picker) — Phase 4.
- Per-car automatic theme selection.
- Dark/light system-preference toggle (existing shadcn theming handles that separately).
- Backend persistence of the selected theme.

## Key Design Decisions (for feature persona to resolve)
1. **Token set**: which CSS variables to vary per theme? Minimum set: `--primary`, `--primary-foreground`,
   `--background`, `--foreground`, `--muted`, `--accent`, `--card`. Feature persona to map these to the
   shadcn variable names already in `index.css`.
2. **Swatch design in Settings**: should swatches show the car model icon or just the color chip? Suggest
   color chip only (simpler, no copyright concerns with brand logos).
3. **Apply-on-select vs apply-on-save**: apply immediately (better UX), persist immediately to localStorage
   (no separate save step needed since it's local only).

## Acceptance Criteria
- [ ] 6–8 preset themes defined as CSS variable override objects in a single `lib/themes.ts` file.
- [ ] Selecting a theme in Settings changes the UI colors immediately across all pages.
- [ ] The selected theme persists on page reload (localStorage).
- [ ] Default theme matches the current look exactly (no unintended style regression).
- [ ] Theme selector card is visible in Settings with swatches for each preset.
- [ ] `npx tsc --noEmit` and `npm run build` pass.

## Handoff → feature
Define the theme token map (which CSS variables each theme overrides), the swatch grid layout spec for
the Settings card, and the `ThemeProvider`/`lib/themes.ts` shape. All data/state via ADR-002 patterns
(no `fetch` in components; this is pure local state so no `api.ts`/`hooks.ts` needed — use a
React context or a Zustand-style store in `lib/`). Hand off to developer with the token table and
component layout spec filled in.

## PM Decision (2026-06-03)
High-impact, zero-backend UX win that makes Voltana feel polished and personal. CSS variable theming
requires no migration, no API change, and no third-party dependency. localStorage persistence is
intentionally scoped out of the backend to keep this as a quick ship. Feature persona owns the palette
design and token mapping; developer wires the `ThemeProvider` and Settings card. Reviewer: dev_supervisor.
No security pass (no auth/secret surface).
