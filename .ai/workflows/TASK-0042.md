## ARCHITECTURE REQUIREMENT (binding for the whole task)

**`estimateConsumption(context)` вҖ” one extensible function.** All consumption estimation MUST flow
through a single function that today returns the **car's average kWh/100km** but accepts a `context`
object so later signals plug in **without a rewrite**:

```ts
// voltana-web/src/features/charging/consumption.ts  (or a lib/ equivalent)
type ConsumptionContext = {
  car: { avgKwhPer100km: number; usableCapacityKwh: number };
  month?: number;          // Jalali month (FEAT-2 season multiplier)
  regenFactor?: number;    // FEAT-4
  sameLocationAsPrev?: boolean; // FEAT-4 elevation heuristic
  // future: elevation delta, temperature, payload вҖҰ
};
function estimateConsumption(ctx: ConsumptionContext): number; // kWh/100km
```

- FEAT-1 uses only `car.avg`. FEAT-2 multiplies by the seasonal factor. FEAT-4 applies
  `Г— (1 вҲ’ regenFactor)`. **No feature may fork its own consumption math** вҖ” each adds a term to the
  same function/context. dev_supervisor rejects any slice that duplicates consumption logic.
- **BUG-1 mirror rule for dates:** every date/month rendered anywhere in the app goes through
  `src/lib/dates.ts`. No hardcoded Jalali or Gregorian formatting outside that file. Adding English
  later must flip every date to Gregorian with zero per-component edits.
