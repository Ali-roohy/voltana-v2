## Problem Statement

Admins have no visibility into who has registered on the platform and no way to manage user accounts. There is no UI or API to list users, toggle admin/verified flags, or remove accounts. The `users` table and cascades are already in place — this task adds the management layer on top.

---

## Scope In

- **Backend**: new endpoints under `/v1/admin/users` (all behind `AdminOnly`):
  - `GET /v1/admin/users` — paginated list
  - `GET /v1/admin/users/:id` — single user detail
  - `PUT /v1/admin/users/:id` — update `is_admin` and/or `is_email_verified`
  - `DELETE /v1/admin/users/:id` — delete user + cascade all their data

- **Frontend**: new `/admin/users` page, admin nav link, user table with action controls

## Scope Out

- Impersonation / "login as user"
- Viewing or editing another user's cars / charging sessions / settings
- Admin-initiated password reset (out-of-band — not needed for this phase)
- Soft-delete / audit log
- Bulk actions

## API Contract

### `GET /v1/admin/users?limit=20&offset=0`

Auth: JWT + AdminOnly

Response `200`:
```json
{
  "items": [
    {
      "id": "uuid",
      "email": "user@example.com",       // null for phone-only accounts
      "phone": "+989123456789",           // null if not linked
      "is_admin": false,
      "is_email_verified": true,
      "bale_linked": true,
      "telegram_linked": false,
      "created_at": "2026-06-01T12:00:00Z"
    }
  ],
  "total": 42,
  "limit": 20,
  "offset": 0
}
```

- Default `limit=20`, max `limit=100`. `offset=0` default.
- Ordered by `created_at DESC`.

### `GET /v1/admin/users/:id`

Auth: JWT + AdminOnly

Response `200`: single `UserSummary` object (same shape as list item) — no 404 enumeration, return 404 on unknown ID.

### `PUT /v1/admin/users/:id`

Auth: JWT + AdminOnly

Request body (all fields optional, at least one required):
```json
{
  "is_admin": false,
  "is_email_verified": true
}
```

Guards:
- Cannot set `is_admin=false` on self (would lock out the only admin session) → `400 {"error":"cannot remove own admin"}`
- Cannot delete or demote if they are the last admin → `400 {"error":"cannot remove the last admin"}`

Response `200`: updated `UserSummary`.

### `DELETE /v1/admin/users/:id`

Auth: JWT + AdminOnly

Guards:
- Cannot delete self → `400 {"error":"cannot delete own account from admin panel"}`
- Cannot delete the last admin → `400 {"error":"cannot delete the last admin"}`

Response `204` on success. All user data deleted via existing `ON DELETE CASCADE` FKs (cars, charging_sessions, user_settings, battery_health_snapshots, email_verification_tokens).

## Backend Implementation Plan

### 1. Extend `UserRepository` interface (no migration)

Add to `internal/repository/user_repo.go`:

```go
// ListAll returns all users ordered by created_at DESC with total count.
ListAll(ctx context.Context, limit, offset int) ([]*domain.User, int, error)

// AdminUpdate patches is_admin and/or is_email_verified on a user.
// Non-nil pointer fields are applied; nil fields are left unchanged.
AdminUpdate(ctx context.Context, id uuid.UUID, isAdmin *bool, isEmailVerified *bool) (*domain.User, error)

// Delete hard-deletes a user; all child rows cascade via FK.
Delete(ctx context.Context, id uuid.UUID) error

// CountAdmins returns the number of users with is_admin=true.
CountAdmins(ctx context.Context) (int, error)
```

Implementation note:
- `ListAll` uses `SELECT … ORDER BY created_at DESC LIMIT $1 OFFSET $2` plus a separate `SELECT COUNT(*) FROM users`.
- `AdminUpdate` uses `UPDATE users SET is_admin=COALESCE($2,is_admin), is_email_verified=COALESCE($3,is_email_verified) WHERE id=$1 RETURNING …`.
- `Delete` uses `DELETE FROM users WHERE id=$1`.
- `CountAdmins` uses `SELECT COUNT(*) FROM users WHERE is_admin=true`.

### 2. New `AdminService` (`internal/service/admin_service.go`)

```go
type AdminService struct {
    users UserRepository
}

func NewAdminService(users UserRepository) *AdminService

// ListUsers paginates; validates limit (max 100).
func (s *AdminService) ListUsers(ctx, limit, offset int) ([]*domain.User, int, error)

// GetUser fetches a single user or returns ErrNotFound.
func (s *AdminService) GetUser(ctx, id uuid.UUID) (*domain.User, error)

// UpdateUser patches flags. Guards:
//   - setting isAdmin=false on self → ErrRemoveSelfAdmin
//   - removing last admin → ErrLastAdmin
func (s *AdminService) UpdateUser(ctx, callerID, targetID uuid.UUID, isAdmin *bool, isEmailVerified *bool) (*domain.User, error)

// DeleteUser hard-deletes. Guards:
//   - deleting self → ErrDeleteSelf
//   - deleting last admin → ErrLastAdmin
func (s *AdminService) DeleteUser(ctx, callerID, targetID uuid.UUID) error
```

Sentinel errors: `ErrRemoveSelfAdmin`, `ErrLastAdmin`, `ErrDeleteSelf` (map to 400 in handler).

### 3. Extend `AdminHandler` (`internal/handler/admin_handler.go`)

Add `*service.AdminService` as a second field on `AdminHandler`:

```go
type AdminHandler struct {
    auth  *service.AuthService
    admin *service.AdminService
}
func NewAdminHandler(auth *service.AuthService, admin *service.AdminService) *AdminHandler
```

Add 4 handler methods:
- `ListUsers(c *gin.Context)`
- `GetUser(c *gin.Context)`
- `UpdateUser(c *gin.Context)`
- `DeleteUser(c *gin.Context)`

All read the caller's `uuid.UUID` from `middleware.UserIDKey`.

### 4. Wire routes in `cmd/server/main.go`

```go
adminSvc := service.NewAdminService(userRepo)
adminH   := handler.NewAdminHandler(authSvc, adminSvc)

// (existing)
v1.POST("/admin/test-otp", middleware.AdminOnly(authSvc), adminH.TestOTPDelivery)

// (new)
v1.GET   ("/admin/users",     middleware.AdminOnly(authSvc), adminH.ListUsers)
v1.GET   ("/admin/users/:id", middleware.AdminOnly(authSvc), adminH.GetUser)
v1.PUT   ("/admin/users/:id", middleware.AdminOnly(authSvc), adminH.UpdateUser)
v1.DELETE("/admin/users/:id", middleware.AdminOnly(authSvc), adminH.DeleteUser)
```

## Frontend Implementation Plan

```ts
export interface UserSummary {
  id: string;
  email: string | null;
  phone: string | null;
  is_admin: boolean;
  is_email_verified: boolean;
  bale_linked: boolean;
  telegram_linked: boolean;
  created_at: string;
}
export interface UsersPage { items: UserSummary[]; total: number; limit: number; offset: number; }

export async function listAdminUsers(limit = 20, offset = 0): Promise<UsersPage>
export async function updateAdminUser(id: string, patch: { is_admin?: boolean; is_email_verified?: boolean }): Promise<UserSummary>

```

**`hooks.ts`**: TanStack Query wrappers:
- `useAdminUsers(limit, offset)` — queryKey `["admin","users", limit, offset]`
- `useUpdateAdminUser()` — mutation; on success invalidates `["admin","users"]`
- `useDeleteAdminUser()` — mutation; on success invalidates `["admin","users"]`

## Acceptance Criteria

1. `GET /v1/admin/users` returns paginated list (non-admin → 403, unauthenticated → 401).
2. `PUT /v1/admin/users/:id` with `{"is_admin":true}` grants admin role; subsequent `GET /v1/me` for that user returns `is_admin:true`.
3. `PUT /v1/admin/users/:id` with `{"is_admin":false}` on self → `400 {"error":"cannot remove own admin"}`.
4. `DELETE /v1/admin/users/:id` on an existing non-admin user → `204`; user no longer appears in list.
5. `DELETE /v1/admin/users/:id` on self → `400`.
6. `DELETE` on last remaining admin → `400`.
7. `GET /v1/admin/users` by a non-admin JWT → `403`.
8. Frontend `/admin/users` page renders user table with badges; rows show correct linked/verified state.
9. Delete confirm dialog present; clicking "Delete" removes the row; toast confirms.
10. Admin nav link only visible when `me.is_admin === true`.
11. `go test ./...` passes · `tsc --noEmit` 0 errors · `npm run build` succeeds.
