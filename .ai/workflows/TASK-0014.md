## Background (why this is READY now)
Across the last three backend closes, qa could not redeploy via `docker compose` and had to:
- host-compile a binary and `docker run` a one-off `Dockerfile.runtime` image,
- manually `nginx -s reload` because nginx **caches the upstream `api` IP at startup** and kept routing to a
  stale/duplicate container (`voltana-api-new`),
- flip `is_email_verified` directly in Postgres to get a JWT (no SMTP catcher to read the verify link).
This task fixes the deploy path itself so future tasks stop paying that tax.

## Scope In
1. **Reproducible `docker compose` redeploy** ‚Äî a rebuilt api image comes up cleanly with `docker compose up
   -d --build api` (or documented equivalent). Resolve the **compose-v1 `KeyError: 'ContainerConfig'`** quirk
   noted in context.md ‚Äî install/standardize on the **Compose v2 plugin** on dev + VPS, and document the
   redeploy command in the QA runbook. (The wedge-prone in-container `golang:1.22-alpine` build vs the
   host-binary `Dockerfile.runtime` path: pick one and document it.)
2. **nginx re-resolves the api on restart** ‚Äî so swapping/restarting the api container does not strand nginx
   on a stale IP. Use a `resolver` (Docker DNS `127.0.0.11`) + a **variable `proxy_pass`** (e.g.
   `set $upstream http://api:9090; proxy_pass $upstream;`) so nginx re-resolves per request, or an equivalent
   that survives an api recreate without a manual `nginx -s reload`.
3. **MailHog (or equivalent) for local SMTP** ‚Äî a dev mail catcher service in compose so registration verify
   links are viewable, removing the manual `is_email_verified` DB flip in smokes. Point `SMTP_HOST` at it in
   the dev `.env`.
4. **`docker-compose.yml` api env** ‚Äî pass **`APP_URL`** + **`SMTP_HOST/PORT/USER/PASSWORD/FROM`** to the api
   service (currently absent ‚Äî a clean `compose up` ships an empty verify link / no SMTP). Set
   **`APP_ENV=production`** for the VPS deploy profile so the refresh cookie gets `Secure` (carry-forward N1);
   keep dev as `development`. No secrets committed ‚Äî placeholders in `.env.example`.
5. **SOH lower-bound floor (1-line code fix, developer)** ‚Äî `internal/service/analytics_service.go:267` clamps
   SOH on top (`estimated > nominal`) but not the bottom; a sub-0.001-kWh qualifying session could round
   `soh_pct` to `0.00` and trip the DB `CHECK (soh_pct > 0)` on Save. Add a lower floor (e.g. clamp `estimated`
   so `soh_pct >= 0.01`) **or** relax the migration CHECK to `>= 0`. Add a unit test for the floor. (Carry-
   forward from TASK-0007.)
