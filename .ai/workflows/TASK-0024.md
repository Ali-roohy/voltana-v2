# TASK-0024 — BUG: No Back/Close Button in Charging Session Detail

**Status**: DONE ✅ CLOSED (qa_supervisor 2026-06-03)
**Phase**: 3 (bug fix)
**Assigned to**: developer
**Reviewer**: dev_supervisor
**Persona**: pm (this spec) → developer (fix) → dev_supervisor → qa
**Depends on**: TASK-0012 (✅ tap-to-expand session detail accordion)

## Bug Description
When a user taps a charging session row to expand the detail accordion, there is **no visible back
arrow, chevron, or close button** to collapse it again. The only way to close the expanded row is to
tap the session header area again — but this affordance is not communicated to the user (no visual cue,
no standard UI indicator). On mobile this is especially confusing because the detail content can fill
most of the screen.

---

## Root Cause (confirmed by pm review of `pages/Charging.tsx`)

`expandedId` state drives the accordion open/close. The session summary row has `aria-expanded` but
no visible **chevron icon** or **close button** to signal the expand/collapse action. The tappable area
is the full row, which works but is undiscoverable — especially on first use.

```tsx
// Charging.tsx — current summary row (simplified)
<div
  onClick={() => setExpandedId(isExpanded ? null : session.id)}
  aria-expanded={isExpanded}
>
  {/* ... session summary cells ... */}
  {/* ← NO chevron, no close button */}
</div>
```

## Fix

**Add a `ChevronDown`/`ChevronUp` icon** to the session summary row that rotates 180° when expanded:

```tsx
import { ChevronDown } from "lucide-react";

// In the summary row:
<ChevronDown
  className={cn(
    "w-4 h-4 text-muted-foreground transition-transform",
    isExpanded && "rotate-180"
  )}
/>
```

**Placement**: rightmost cell of the summary row (or leftmost in RTL layout — developer to verify
the current row layout direction and place it at the trailing edge of the row).

**Additionally**, inside the expanded detail section, add a small **"بستن" (Close) button** or an ×
icon at the top-right corner of the detail panel as a secondary affordance, for users who have
scrolled the header out of view:

```tsx
<Button
  variant="ghost"
  size="icon"
  className="absolute top-2 left-2"   // adjust for RTL
  onClick={() => setExpandedId(null)}
>
  <X className="w-4 h-4" />
</Button>
```

## Scope
- Summary row: add chevron that rotates on expand.
- Detail panel: add a close button (× or "بستن") at the top of the expanded content.
- No state changes, no API changes, no migration.
- RTL/LTR positioning: chevron at the trailing edge (logical `end`), close button at the top of the
  detail area.

## Acceptance Criteria
- [ ] A chevron icon is visible on each session row; it points down when collapsed and up when expanded.
- [ ] Tapping the close button (× / "بستن") inside the detail panel collapses the row.
- [ ] Existing tap-to-expand behavior is unchanged.
- [ ] RTL layout is respected: chevron and close button are on the correct edge.
- [ ] `npx tsc --noEmit` and `npm run build` pass.

## Handoff → developer

close button handles the case where the user has scrolled down into the detail. Priority: **READY**.
