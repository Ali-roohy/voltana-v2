## Goal

Surface the time-of-use (TOU) split that Voltana already stores — peak / mid /
off-peak energy and cost — as a visual **stacked breakdown** so users can see *where*
their charging spend goes, not just a single total. Render it in two places: the
**dashboard** (aggregate across the selected scope) and **each session card** (per-session
split). High impact, low effort — **no API or DB change**; all data already exists.

Source of the proposal: researcher report, top-3 (High impact / Low effort).

---

## Background / Grounding (already in the codebase)

- Each session (`features/charging/api.ts → ChargingSession`) carries
  `energy_peak_kwh`, `energy_mid_kwh`, `energy_offpeak_kwh`, `kwh_charged`, and `cost`.
- Per-period **rates** come from settings (`features/settings/api.ts → Settings`:
  `peak_rate`, `mid_rate`, `offpeak_rate`).
- Cost logic already exists in `pages/Charging.tsx` as `getSessionCost()` —
  `session.cost ?? (peak·peak_rate + mid·mid_rate + offpeak·offpeak_rate)`. This task
  should **reuse / extract** that logic, not re-derive it inline a third time.
- `recharts` (^2.15.4) is already a dependency and used on the dashboard (`pages/Index.tsx`).

---

## Scope In

- **Shared component** — a single reusable breakdown component (e.g.
  `src/components/TouCostBreakdown.tsx`) that takes peak/mid/off-peak **kWh and cost**
  and renders a stacked bar with three segments (peak, mid, off-peak), each labelled with
  its kWh and cost, plus the total.
- **Per-session card** (`pages/Charging.tsx`) — show the breakdown for that session's
  three energy buckets × the user's rates. Hide segments that are zero. Degrade gracefully
  when a session has no per-period split (only `kwh_charged`) — show total only, no crash.
- **Dashboard** (`pages/Index.tsx`) — show an **aggregate** breakdown across the dashboard's
  current scope (it already filters by `default_car_id`): sum peak/mid/off-peak kWh and cost
  across sessions and render the same component as a summary card.
- **Cost helper extraction** — move the per-period cost math into a single shared helper
  (suggest `features/charging/cost.ts` or `src/lib/`) used by `getSessionCost`, the session
  card, and the dashboard aggregate, so there is one source of truth.
- **i18n + RTL** — Persian/English labels (peak = اوج بار, mid = میان باری, off-peak =
  کم باری) and correct rendering under `dir="rtl"`, consistent with existing cards.
- **Currency unit** — match the unit already displayed on the Charging page (`formatCost`
  currently renders **ریال / Rial**). See Open Question below before introducing "Toman".

## Scope Out

- Any API/DB/migration change (data already exists — explicitly **none**).
- Editing or overriding per-period cost (the inline cost override already exists; untouched).
- Monthly trend over time — that is **TASK-0011**.
- Date-range filtering of which sessions feed the dashboard aggregate — that is **TASK-0012**.

## Feature Design (feature persona — 2026-06-01) → hand to developer

### Grounding (verified against current code)
- `ChargingSession` (`features/charging/api.ts`): `energy_peak_kwh|energy_mid_kwh|energy_offpeak_kwh|kwh_charged|cost` are all `number | null`.
- `Settings` (`features/settings/api.ts`): `peak_rate|mid_rate|offpeak_rate: number`.
- `pages/Charging.tsx`: local `rates = {peak,mid,offpeak}` (`?? 0`), `getSessionCost(session) = session.cost ?? Σ(energy·rate)`, `totalKwh(session) = kwh_charged ?? Σ(energy)`, `formatCost = Intl.NumberFormat("fa-IR")`, and the displayed cost line is `{formatCost(getSessionCost(session))} ریال` (~L422). Per-period **kWh** are already shown as sub-lines under the Zap block (~L378–391).
- `pages/Index.tsx`: `stats` memo scopes by `default_car_id`, sorts by `started_at`, and currently computes `totalCost` as **`Σ (s.cost ?? 0)`** — this **undercounts** sessions with no stored `cost` (no rate fallback). The helper extraction fixes this (see C3).
- Number formatting: `formatNumber` exists in `src/lib/utils.ts` (fa-IR grouping). i18n lives in `src/i18n/locales/{en,fa}.json` (existing `charging` namespace).

### 1) Shared helper — `src/lib/cost.ts`
Single source of truth for TOU cost. Pure functions, no React, no fetch.

```
type Rates = { peak: number; mid: number; offpeak: number }
type TouCost = { peak: number; mid: number; offpeak: number; total: number }

ratesFromSettings(settings?: Settings): Rates
  → { peak: settings?.peak_rate ?? 0, mid: …mid_rate ?? 0, offpeak: …offpeak_rate ?? 0 }

calcCost(session: Pick<ChargingSession,
           "energy_peak_kwh"|"energy_mid_kwh"|"energy_offpeak_kwh">,
         rates: Rates): TouCost
  peak    = (energy_peak_kwh    ?? 0) * rates.peak
  mid     = (energy_mid_kwh     ?? 0) * rates.mid
  offpeak = (energy_offpeak_kwh ?? 0) * rates.offpeak
  total   = peak + mid + offpeak          // segments ALWAYS sum to total (parity)
```

**Manual-override handling (important):** the inline per-session cost override (`session.cost`) is a
single number with no period split, so it stays a **call-site** concern, not inside `calcCost`.
`getSessionCost` becomes:  `session.cost ?? calcCost(session, rates).total`. This preserves today's
exact behavior **and** keeps `calcCost` segments summing to `total`. (So for an overridden session the
breakdown bar shows the rate-based split while the headline cost shows the override — documented, rare.)

### 2) Reusable component — `src/components/TOUBreakdown.tsx`
**Presentational only** (no `ChargingSession`/hooks coupling) so it serves both the per-session card and
the dashboard aggregate. Lightweight CSS stacked bar.

```
type TouSegment = { kwh: number; cost: number }
interface TOUBreakdownProps {
  peak: TouSegment; mid: TouSegment; offpeak: TouSegment;
  total?: TouSegment;        // default = sum of the three segments
  variant?: "inline" | "summary";   // inline = per-session (compact); summary = dashboard card
  className?: string;
}
```
- **Bar:** one horizontal flex row; each segment width ∝ its **cost** share of the total (where the money
  goes). Colors — **peak = red (`bg-red-500`), mid = amber (`bg-amber-500`), off-peak = green
  (`bg-green-500`)** (+ existing dark-mode conventions). Round the bar ends; thin (`h-2`/`h-3` inline,
  taller for summary).
- **Zero segments omitted** — a segment with `kwh === 0` (or `cost === 0`) renders no slice and no legend row.
- **Legend / rows:** for each present segment, label (i18n) + `formatNumber(kwh)` kWh + `formatNumber(cost)` **تومان**.
- **Total row:** total kWh + total تومان.
- **Degraded state:** if all three segment kWh are 0 but `total.kwh > 0` (session has only `kwh_charged`,
  no split) → render **no bar**, just the total row + a muted "بدون تفکیک" / "No breakdown" hint. No crash.
- **i18n + RTL:** all labels via i18n keys (below). Bar must fill from the right under `dir="rtl"`; verify
  segment reading order peak→mid→offpeak in both directions (use a plain flex row and let global `dir`
  handle it; check visually).

### 3) Wiring
**A — `pages/Charging.tsx` (per-session):**
1. `import { calcCost, ratesFromSettings } from "@/lib/cost"`; replace the local `getSessionCost` body
   with `session.cost ?? calcCost(session, ratesFromSettings(settings)).total` (delete the inline rate math).
2. Mount `<TOUBreakdown variant="inline" …>` inside each session `Card` (near the existing cost line):
   pass per-segment `{ kwh: session.energy_*_kwh ?? 0, cost: calcCost(...).<seg> }` and
   `total={{ kwh: totalKwh(session), cost: getSessionCost(session) }}`.
3. **Relabel** the existing cost line `ریال` → `تومان` (decision #1). Leave `formatCost`/override UI as-is.

**B — `pages/Index.tsx` (dashboard summary, current month):**
1. `const rates = ratesFromSettings(settings)`.
2. In the `stats` memo add a **current-month** aggregate: `const ym = new Date().toISOString().slice(0,7)`;
   over the already-scoped `sorted` sessions where `started_at.slice(0,7) === ym`, sum per segment
   `{kwh, cost}` (kwh from `energy_*_kwh ?? 0`, cost from `calcCost(s, rates)`) → `touMonth`.
   (Gregorian `YYYY-MM`, consistent with the existing `energyTrend` bucketing — a Jalali-month variant is
   a later option, not a bug.)
3. **Fix `totalCost`** to use the helper: `Σ (s.cost ?? calcCost(s, rates).total)` instead of `Σ (s.cost ?? 0)`
   — intentional correctness fix (today's number undercounts). Flag in the PR for dev_supervisor.
4. Render a **"این ماه" / "This month"** summary `Card` with `<TOUBreakdown variant="summary" {...touMonth}>`
   near the existing stat cards / energy chart. Empty month (no sessions) → component shows the degraded
   total-only/zero state cleanly.

**C — i18n (`src/i18n/locales/{en,fa}.json`):** add a `tou` group — `title`/`thisMonth`,
`peak` (اوج بار / Peak), `mid` (میان باری / Mid), `offpeak` (کم باری / Off-peak),
`toman` (تومان / Toman), `total` (مجموع / Total), `noBreakdown` (بدون تفکیک / No breakdown).

### Parity / acceptance notes for the developer
- `calcCost` segments **always** sum to `calcCost().total` → satisfies the "segments sum to total" check.
- `getSessionCost == session.cost ?? calcCost().total` → unchanged displayed per-session cost (parity).
- One source of truth: after this, the rate math exists **only** in `lib/cost.ts` (none inline in
  `Charging.tsx` or `Index.tsx`).
