### Final `charging_stations` schema (000007 up)
```sql
-- Shared reference data (NOT user-owned) — no user_id, readable by every authed user.
-- Mutable (admin edits), so it carries updated_at + the shared set_updated_at() trigger,
-- unlike the read-only ev_models reference table.
CREATE TABLE charging_stations (
    id              UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    name            VARCHAR(255) NOT NULL,
    latitude        DOUBLE PRECISION NOT NULL CHECK (latitude  >= -90  AND latitude  <= 90),
    longitude       DOUBLE PRECISION NOT NULL CHECK (longitude >= -180 AND longitude <= 180),
    address         VARCHAR(500),
    connector_types VARCHAR(255),   -- CSV e.g. "CCS2,Type2" (normalize to a child table only if needed later)
    power_kw        INT          CHECK (power_kw IS NULL OR power_kw > 0),
    operator        VARCHAR(255),
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT now(),
    updated_at      TIMESTAMPTZ  NOT NULL DEFAULT now()
);

CREATE TRIGGER charging_stations_updated_at
    BEFORE UPDATE ON charging_stations
    FOR EACH ROW EXECUTE FUNCTION set_updated_at();
```
- DB-level lat/lng `CHECK`s back the service-level bounds validation (defense in depth, mirrors the
  `start_soc`/`end_soc` CHECKs on charging_sessions).
- No `status`/`availability`/`price` columns — explicitly Scope Out. The frontend status badges must be
  removed (see §E), not faked.
- No index needed yet (small bounded reference table); a bbox/`GiST` index is a future follow-up if station
  counts grow — noted, not in scope.

## B. Admin authorization — `users.is_admin` + `AdminOnly` middleware (fresh DB check)
- Add `IsAdmin bool` to `domain.User`; extend `userCols` in `repository/user_repo.go` to
  `..., is_admin, ...` and the `scanUser` arg list accordingly (one line each). `FindByID` already exists.
- **The admin flag is checked against the DB on each write, NOT carried in the JWT.** Rationale: the access
  token lives 15 min; baking `is_admin` into it means a revoked admin keeps write power for up to 15 min.
  Station writes are rare, so one `FindByID` per write is cheap and gives **immediate** revocation. This is
  the security-correct choice and what the security reviewer should expect.
- New `internal/middleware/admin.go`:
  ```go
  // AdminOnly must run AFTER Auth (which sets UserIDKey). It 403s any non-admin.
  func AdminOnly(authSvc *service.AuthService) gin.HandlerFunc
  ```
  It reads `c.MustGet(UserIDKey).(uuid.UUID)`, calls a new `authSvc.IsAdmin(ctx, userID) (bool, error)`,
  and on `false` aborts with `403 {"error":"admin privileges required","code":"FORBIDDEN"}`; on lookup
  error → `500 {"...","code":"INTERNAL"}`. Mirrors the abort style in `middleware/auth.go`.
- New `AuthService.IsAdmin(ctx, userID)` delegates to `s.users.FindByID` and returns `u.IsAdmin` (keeps the
  middleware→service dependency direction the existing `Auth` middleware already uses; middleware never
  imports `repository`).

## C. API contract — `/v1/stations` (all under the `v1` Auth group)
| Method | Path | Authz | Success | Body / Notes |
|---|---|---|---|---|
| GET | `/v1/stations` | any authed | 200 `listResponse` | **markers** — see field split. Returns the full set (see pagination note). |
| GET | `/v1/stations/:id` | any authed | 200 station detail | full row. `:id` malformed → 400 `INVALID_REQUEST`; missing → 404 `NOT_FOUND`. |
| POST | `/v1/stations` | **admin** | 201 station detail | create. Non-admin → 403 `FORBIDDEN`. |
| PUT | `/v1/stations/:id` | **admin** | 200 station detail | full replace (same body as POST). |
| DELETE | `/v1/stations/:id` | **admin** | 204 no content | non-admin → 403; missing → 404. |

- **Marker vs detail field split:**
  - **Marker** (GET list item): `id, name, latitude, longitude, connector_types, power_kw`. Enough to plot +
    label; keeps the map payload light.
  - **Detail** (GET `:id`, and the create/update/POST/PUT responses): all columns —
    `id, name, latitude, longitude, address, connector_types, power_kw, operator, created_at, updated_at`.
- **Pagination decision:** the map needs **every** marker, so `GET /v1/stations` returns the full set, not a
  100-row page. Still wrap it in the standard `listResponse{items,limit,offset,total}` envelope for
  consistency, with `limit == total`, `offset == 0`. (A bbox/paged variant is a documented future follow-up;
  fine while station count is small.) `:id`/detail and writes are single-object, no envelope.
- **Request body (POST/PUT)** — `stationRequest` binding, mirroring `carRequest`:
  ```go
  type stationRequest struct {
      Name           string  `json:"name"            binding:"required,max=255"`
      Latitude       float64 `json:"latitude"        binding:"required,min=-90,max=90"`
      Longitude      float64 `json:"longitude"       binding:"required,min=-180,max=180"`
      Address        *string `json:"address"         binding:"omitempty,max=500"`
      ConnectorTypes *string `json:"connector_types" binding:"omitempty,max=255"`
      PowerKW        *int    `json:"power_kw"        binding:"omitempty,min=1"`
      Operator       *string `json:"operator"        binding:"omitempty,max=255"`
  }
  ```
  Note: gin `binding:"required"` treats `0` as missing — `latitude:0` (the equator) would be rejected by
  binding alone. So **lat/lng bounds + presence are validated in the service** (the binding `min/max` are a
  first cushion); the service re-checks `-90..90` / `-180..180` and returns `service.ErrValidation` →
  400 `INVALID_REQUEST`. Reuse the **existing** `service.ErrValidation` sentinel (defined in
  `car_service.go`); add a new `service.ErrStationNotFound` → 404 `NOT_FOUND`.
- **Error envelope:** reuse `apiError(c, status, code, msg)` and codes already in use:
  `INVALID_REQUEST` (400), `FORBIDDEN` (403, new — admin gate), `NOT_FOUND` (404), `INTERNAL` (500).
  403 must be a clean deny with **no leak** of whether the station exists (security: gate runs before any
  lookup, so a non-admin PUT/DELETE on any id — real or not — returns 403, never 404).
