## FEAT-3 Design (Fable) — Theme Background Designer

**Layering principle**: the background is an independent layer UNDER the theme system, not a 9th
preset. Theme presets / dynamic car-colors keep owning accent vars (`--primary`, …); the
background owns only the page canvas. Two independent storage keys → any combination works.

### Persistence — new key `voltana:bg` (JSON)

```ts
type BackgroundStyle =
  | { type: "solid"; color: string }                                  // hex
  | { type: "gradient"; direction: "to-b"|"to-br"|"to-r"|"to-tr"; stops: string[] } // 2–3 hex
  | { type: "pattern"; id: "dots" | "grid" | "stripes" };
// absence of the key = theme-default background
```

### Mechanism

- All ~10 page wrappers currently hardcode `bg-background` (6) or the
  `bg-gradient-to-br from-background via-secondary to-background` hero gradient (4) →
  replaced by two shared classes `.app-page-bg` / `.app-page-bg-gradient` in `index.css`
  (identical default rendering — zero visual change without an override).
- `lib/background.ts` applies the override: sets `--app-custom-bg` on `:root` +
  `data-custom-bg` attr on `<html>`; one CSS rule makes the page classes transparent and paints
  `body` with `var(--app-custom-bg)` (`background-attachment: fixed`).
- **Composition rule**: `pattern` presets are built from
  `hsl(var(--background))` + `hsl(var(--primary)/α)` — they automatically re-tint when the user
  switches any of the 8 presets or a dynamic car-color theme. `solid`/`gradient` are explicit
  user colors and intentionally theme-independent. Switching themes never touches `voltana:bg`
  and vice versa.

### Picker UI (Settings → «پس‌زمینه» card, below the theme card)

- 4 mode chips: «پیش‌فرض» «رنگ ثابت» «گرادیان» «الگو»
- solid → native color input; gradient → direction select (↓ ↘ → ↗) + 2–3 color stops
  (add/remove third); pattern → 3 self-previewing swatches (نقطه‌ای/شبکه‌ای/راه‌راه)
- Live apply on every change; «پیش‌فرض» clears the key (reset path)
- ThemeProvider applies the saved background on mount (after theme restore)
