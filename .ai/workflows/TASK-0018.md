## Goal
Let users record an **odometer reading** with each charging session so Voltana can compute real driving
efficiency (**kWh/100km**) — per session and as a fleet average — instead of showing a placeholder.
> **Correction (developer, 2026-06-03):** the spec assumed `charging_sessions.odometer_km` already existed.
> It did **not** (the column was only on `cars`). Added as migration **000009** (operator-approved). See the
> Implementation Log.

---

## Scope In
- Add an **optional `odometer_km`** field to the charging-session form.
- Persist to **`charging_sessions.odometer_km`** (column already exists in the DB — verify before adding any
  migration).
- Compute **kWh/100km per session** when an odometer reading is available (and a prior reading exists to
  derive distance).
- Show the per-session efficiency on the **session card**.
- Show a **fleet average efficiency** across all sessions that have the data on the **dashboard**.

## Scope Out
- **OBD auto-fill** of the odometer (Phase 4).
- **Sub-session / GPS per-trip efficiency** (instantaneous trip readings). Per-session efficiency derived from
  two consecutive odometer readings **is in scope** (see the note reconciling this with Acceptance below).

## Acceptance Criteria
- [ ] `odometer_km` is **optional** — existing sessions (and new ones left blank) are unaffected; no
      regression to create/update/list.
- [ ] When two **consecutive** sessions (for the same car, by time) both have an odometer reading, compute
      **`km_driven = odometer_new − odometer_old`**.
- [ ] **`kWh/100km = (session_kwh / km_driven) × 100`** (guard `km_driven > 0`; ignore non-positive/﻿zero).
- [ ] The session card shows **"X kWh/100km"** for sessions where it can be computed.
- [ ] The dashboard **average-efficiency card shows real data** (no longer "—") once odometer data exists.
- [ ] `cd voltana-api && go build ./... && go vet ./... && go test ./...` and
      `cd voltana-web && npx tsc --noEmit && npm run build` all pass.

## Notes / pointers
- **`odometer_km` column already exists in `charging_sessions`** (per the original schema). Developer must
  **confirm on disk** (`\d charging_sessions` / the migration) before assuming — if absent, that's a scope
  change (new migration) to flag back to pm.
- **Reconcile Scope Out vs Acceptance:** "per-trip efficiency" in Scope Out means *sub-session/GPS* readings,
  which are out. The Acceptance formula (distance from two **consecutive session** odometer readings) is the
  in-scope per-session metric — implement that.
- **Dashboard avg-efficiency surface:** TASK-0008 added `avg_kwh_per_100km` (null when `total_km == 0`) and
  TASK-0011 repurposed the dashboard's old avg-efficiency *card* → avg cost/session. Developer to confirm the
  current dashboard wiring and decide whether to (a) revive the efficiency value on an existing card or
  (b) surface it where the analytics API already exposes it — without breaking the 0011 cost card. Keep the
  computation server-side in the analytics service where the totals already live.
- Per-session efficiency attributes a session's added energy to the distance driven since the **previous**
  odometer reading (a standard approximation); compute it where the session list is assembled (service layer),
  not in the component.

# Implementation Log — developer (2026-06-03)

## ⚠️ Spec premise was wrong (operator-approved fix): added migration 000009
The spec stated `odometer_km` "already exists in charging_sessions". **It does not** — `\d charging_sessions`
confirmed the column is absent; `odometer_km` lives only on the **`cars`** table. Per the spec's own
"verify before assuming / flag back to pm" note and the Auto-Chain "pause for blocker" rule, this was raised;
operator chose **add an additive nullable column** (migration **000009**). Consequence: TASK-0017's contract
reserved `000009` — it shifts to **000010** (next implementer takes the next number).

## Changes
**Migration** `000009_charging_sessions_odometer` — `ADD COLUMN odometer_km INT` (nullable, no default) +
`CHECK (odometer_km IS NULL OR >= 0)`; down drops both. Existing rows → NULL (unaffected).

**Backend**
- `domain.ChargingSession`: `OdometerKM *int` (`odometer_km`), transient `PrevOdometerKM *int` (`json:"-"`),
  derived `EfficiencyKWhPer100km *float64` (`efficiency_kwh_per_100km`). `ChargingInput.OdometerKM`.
- `charging_handler`: `odometer_km` request field (`binding:"omitempty,min=0"`) + `toInput`.
- `charging_repo`: `odometer_km` in cols/INSERT/UPDATE/scan; **List** adds
  `LAG(odometer_km) OVER (PARTITION BY car_id ORDER BY started_at)` → `PrevOdometerKM` (window runs over the
  WHERE set before LIMIT, so it's correct across pages); new `EfficiencyAggregateByUser` (CTE summing energy +
  positive odometer deltas over consecutive logged sessions).
- `charging_service`: `setSessionEfficiency` in `List` (kwh / (Δkm/100), only when both readings + energy +
  Δ>0); odometer `>= 0` validation.
- `analytics_service.GetDashboard`: `AvgKWhPer100KM` now derived from `EfficiencyAggregateByUser` (session
  odometer deltas) instead of the cars'-odometer ratio. `TotalKM` unchanged (still car-odometer sum).
- Tests: `TestCharging_PerSessionEfficiency`, `TestCharging_EfficiencyNilWhenOdometerNotIncreasing`,
  updated `TestAnalytics_DashboardAggregates` (now session-odometer-derived, avg 8.0); mock `ListByUser` mirrors
  the LAG so the service calc is exercised.

**Frontend**
- `features/charging/api.ts`: `odometer_km` + `efficiency_kwh_per_100km` on `ChargingSession`; `odometer_km?`
  on `ChargingInput`.
- `Charging.tsx`: optional odometer form field; `buildInput`/`handleEdit` wired; **inline cost-save now
  round-trips `odometer_km`** (PUT is full-replace — would otherwise wipe it); session card shows
  "X kWh/100km" when available.
- Dashboard `Index.tsx` **unchanged** — it already renders `avg_kwh_per_100km` (shows real data now).

## Deviation
- API field is **`power_kw`-style `odometer_km`** as specified, but the spec's "kWh/100km" attribution model:
  a session's energy is attributed to the distance since the **previous** session's odometer (the Acceptance
  formula). "per-trip" in Scope Out = GPS/sub-session, correctly excluded.

## Evidence
- API (host): `go build ./...` ✓ · `go vet ./...` ✓ · `go test ./...` ✓ (incl. 3 new/updated tests).
- Frontend: `tsc --noEmit` 0 · `npm run build` ✓.
