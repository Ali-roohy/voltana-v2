## Goal

The dashboard already plots a **monthly energy (kWh) trend**; add the money dimension
beside it — a **monthly cost trend** (bar/line) plus headline figures **total spend** and
**average cost per session**. This answers "what am I actually spending, and is it trending
up?" High impact, low effort — **no API or DB change**; cost is derived from data already
fetched.

Source of the proposal: researcher report, top-3 (High impact / Low effort).

---

## Background / Grounding (already in the codebase)

- `pages/Index.tsx` already computes a `stats` memo with `energyTrend`
  (`[{ month: "YYYY-MM", energy }]`), `totalEnergy`, and `totalCost`, and renders charts
  via `recharts` (`LineChart`, `BarChart`, `ResponsiveContainer`, etc. already imported).
- Monthly bucketing pattern already exists there: `s.started_at.slice(0, 7)` → `YYYY-MM`.
- Cost per session is `session.cost ?? Σ(energy_period · rate_period)` — same logic as
  `getSessionCost` in `pages/Charging.tsx`. This task should consume the **shared cost
  helper introduced by TASK-0010** rather than re-deriving the formula.
- The dashboard already scopes by `default_car_id`; the cost trend must respect the same
  scope so energy and cost charts line up.

---

## Scope In

- **Monthly cost series** — extend the `Index.tsx` `stats` memo to produce a
  `costTrend` (`[{ month: "YYYY-MM", cost }]`) bucketed by month, summing per-session cost
  over the same scoped+sorted sessions used for `energyTrend`, so the two series share an
  x-axis domain.
- **Chart** — render the cost trend as a bar (or line) chart **beside / below** the existing
  energy chart, using the already-imported `recharts` primitives and matching the existing
  chart styling, tooltip, and RTL handling.
- **Headline figures** — display **total spend** (sum of per-session cost across scope) and
  **average cost per session** (total spend ÷ session count; show "—" / zero-safe when there
  are no sessions).
- **Empty / single-month states** — render cleanly with 0 sessions and with a single month
  (no NaN, no broken axis).
- **i18n + RTL** — Persian/English labels and axis/tooltip formatting consistent with the
  existing energy chart; reuse `formatNumber` / locale number formatting already in `src/lib`.
- **Currency unit** — same unit decision as TASK-0010 (see its Open Question). Do not mix
  units between this chart and the breakdown card.

## Scope Out

- Any API/DB/migration change (**none** — derived client-side from existing session data).
- Per-period (peak/mid/off-peak) stacking inside the trend — the TOU split is **TASK-0010**;
  this chart is total-cost-per-month.
- User-selectable date range / window for the chart — that is **TASK-0012** (this task uses
  the dashboard's existing scope and whatever sessions the list hook already returns).
- Cost forecasting / projections.

## Feature Design (feature persona — 2026-06-01) → hand to developer

### Grounding (verified against current `pages/Index.tsx`, post-TASK-0010)
- The `stats` memo already has: `const rates = ratesFromSettings(settings)`, a scoped+sorted
  `sorted` array (`default_car_id` filter), `totalEnergy`, **`totalCost = Σ (s.cost ?? calcCost(s, rates).total)`**
  (the exact per-session cost expression to reuse), a month-bucket loop
  (`monthly[s.started_at.slice(0,7)] += totalKwh(s)` → `energyTrend: [{month, energy}]`), `socData`, and `touMonth`.
  Memo deps already include the three rate fields. `recharts` `LineChart/Line/BarChart/Bar/XAxis/YAxis/
  CartesianGrid/Tooltip/Legend/ResponsiveContainer` are **all already imported**; so is `formatNumber`.
- **Total spend is already on screen:** the `totalCost` stat card renders `{formatNumber(stats.totalCost)} تومان`.
- **The `avgEfficiency` stat card is dead** — it hardcodes `— kWh/100km` (per-session odometer was dropped).
  Ideal slot to repurpose for **avg cost / session** (no layout change; removes a permanently-"—" metric).
- The "Sessions" stat card counts **unscoped** `sessions.length` (with a hacky `stats.socData.length ?` guard) —
  a minor pre-existing inconsistency vs the scoped `totalCost` (see optional cleanup below).

### Decision — Q3: chart type → **two separate single-unit charts, side by side (NOT dual-axis)**
Render a **new monthly-cost BarChart beside the existing monthly-energy LineChart**, each with its own
single-unit Y-axis. **Rejected dual-axis** because kWh and Toman are unrelated scales — a shared
dual-axis chart invites misreading (the two axes are arbitrary), needs `ComposedChart` (a new import) and
an awkward right-hand axis under RTL. Two clean charts match the existing energy/SOC pattern, reuse the
already-imported `BarChart`/`Bar`, and the researcher framing was literally "beside the existing energy chart."

### 1) Monthly cost series (Q1 — extend the existing trend)
In the `stats` memo, **augment the existing month-bucket loop** to also accumulate cost per month, and
emit a **single shared `trend` array** so both charts share an identical x-domain:
- Per session, cost = `s.cost ?? calcCost(s, rates).total` (reuse — do **not** re-inline the formula).
- Bucket by `s.started_at.slice(0,7)` (same key as energy), producing `trend: [{ month, energy, cost }]`
  sorted by month. (Rename `energyTrend` → `trend` and add `cost`; the energy chart just changes its
  `data={stats.trend}` and keeps `dataKey="energy"`, the new cost chart reads the same array with
  `dataKey="cost"`.) This guarantees the two charts line up month-for-month.

### 2) Headline figures (Q2)
- **Total spend** — already shown by the `totalCost` card; leave it (it's `totalCost` in تومان). No change
  beyond confirming the label reads as total spend.
- **Avg cost / session** — **repurpose the dead `avgEfficiency` card**. In the memo add
  `sessionCount = sorted.length` (scoped) and `avgCost = sessionCount > 0 ? totalCost / sessionCount : null`.
  Render `{avgCost == null ? "—" : formatNumber(Math.round(avgCost)) + " تومان"}`. Zero-safe (no NaN/÷0).
  Swap the card label from `dashboard.avgEfficiency` to a new `dashboard.avgCostPerSession` key.

### 3) Chart + layout
- New **Monthly Cost** card mirrors the energy card exactly (CardHeader gradient + CardTitle w/ icon,
  `ResponsiveContainer height={250}`, `CartesianGrid`, `XAxis dataKey="month"`, `YAxis`, `Tooltip` with
  `formatNumber(...)` + " تومان", `Legend`, a `Bar dataKey="cost"` with a gradient fill). Use a **distinct
  color** from the energy line (e.g., an accent/green gradient like the SOC chart's `colorBar`).
- **Layout:** put the two monthly trends together in the first charts row (`lg:grid-cols-2`): **Energy (line)
  + Cost (bar)**. **Move the existing SOC chart** to its own row below (full-width or its own 2-col cell).
- Tooltip/axis numbers via `formatNumber`; optional polish: abbreviate large Toman on the Y-axis (÷1000 "K") —
  not required.

### 4) i18n + states
- Add keys to `src/i18n/locales/{en,fa}.json` `dashboard` group: `avgCostPerSession`
  (en "Avg Cost / Session" · fa "میانگین هزینه هر جلسه") and `monthlyCost` (en "Monthly Cost (Toman)" ·
  fa "هزینه ماهانه (تومان)"). (Chart titles may also follow the existing inline-ternary style if preferred,
  but i18n keys are cleaner.)
- **Empty (0 sessions):** `trend` empty → chart renders empty axes (as the energy chart already does);
  `avgCost` → "—"; `totalCost` → "0 تومان". **Single month:** one bar — fine. No NaN anywhere.
- **Currency:** Toman, same as TASK-0010 — **no ÷10**; don't mix units with the TOU card.

### Optional cleanup (note for developer / dev_supervisor, not required)
- The "Sessions" card's unscoped `sessions.length` could be switched to the scoped `sessionCount` for
  consistency with `totalCost`/`avgCost`. Small, pre-existing; do it only if trivial.
