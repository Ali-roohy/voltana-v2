p256dh text, auth text, created_at)`. Down drops the table.

### B2 — Backend (lib: `github.com/SherClockHolmes/webpush-go`)
- `VAPID_PUBLIC_KEY` / `VAPID_PRIVATE_KEY` env (dev pair generated into `.env`; placeholders in
  `.env.example`). No keys → push disabled, log notice, endpoints return 503.
- `PushSubscriptionRepository` (Create idempotent-on-endpoint / DeleteByEndpoint / ListByUser / DeleteByID).
- `PushService`: `Subscribe` (validates endpoint: https only, reject localhost/private-IP hosts —
  SSRF guard), `Unsubscribe`, `SendToUser` (fan-out to all subs; **prune on 404/410**).
- Routes: `GET /v1/push/vapid-key` (authed) · `POST /v1/account/push-subscription` ·
  `DELETE /v1/account/push-subscription` (by endpoint) · `POST /v1/admin/test-push` (AdminOnly,
  sends to caller's own subs — mirrors the OTP test panel pattern).
- **Trigger (the one real notification)**: after SOH recompute, if the car's SOH **crosses below
  80%** (prev ≥ 80 → new < 80), push «⚠️ سلامت باتری <car> به <pct>٪ رسید». Optional setter on
  AnalyticsService (nil-safe), same pattern as HealthRecomputer.

### B3 — Frontend
- `sw.js`: `push` (show notification: title/body/url payload, app icon) + `notificationclick`
  (focus/open url).
- `features/push/{api.ts,hooks.ts}`: vapid key fetch, subscribe (PushManager + permission),
  unsubscribe; `urlBase64ToUint8Array` helper.
- Settings card «اعلان‌ها»: enable/disable toggle (permission state aware); hidden when the
  browser lacks `PushManager` or push is disabled server-side (503).

### B4 — Tests
- Unit: subscribe→`SendToUser` delivers an encrypted POST to an `httptest` endpoint (assert hit +
  webpush headers); a 410 response prunes the subscription; endpoint validation rejects
  http/localhost/private hosts; SOH-cross trigger fires exactly on the ≥80→<80 edge.
- qa live: subscription CRUD via curl (fake-but-valid-shape sub), 503-when-unconfigured path or
  configured path per env, migration round-trip.
