## Goal

Stand up the Voltana stack on a production Ubuntu VPS: domain pointing at the server,
HTTPS via Let's Encrypt, hardened firewall, Docker Compose running as a systemd service
with auto-restart, and a one-command deploy script (`deploy.sh`) so the operator can
ship a new version in one step.

---

## Scope In

- [ ] **Server bootstrap script** (`scripts/bootstrap-vps.sh`)
  - Install Docker Engine + Compose v2 plugin
  - Install `certbot` + `python3-certbot-nginx`
  - UFW firewall: allow 22 (SSH), 80, 443; deny all inbound else
  - Create `/opt/voltana/` deploy directory
  - Create `voltana` deploy user (no login shell)

- [ ] **Production nginx config** (`infra/nginx/nginx.prod.conf`)
  - HTTP → HTTPS redirect (301)
  - TLS via Let's Encrypt managed by certbot (or manual cert path)
  - Reverse-proxy to the api container on 9090
  - Reverse-proxy static assets from voltana-web build
  - Security headers: `X-Frame-Options`, `X-Content-Type-Options`, `Strict-Transport-Security`
  - Rate limit zone for `/auth/` endpoints (nginx `limit_req`)

- [ ] **Systemd service** (`infra/systemd/voltana.service`)
  - `ExecStart=docker compose up`
  - `ExecStop=docker compose down`
  - `Restart=always` / `RestartSec=10`
  - Runs as `voltana` user from `/opt/voltana/`
  - `WantedBy=multi-user.target`

- [ ] **Production `.env.example`** additions
  - `APP_ENV=production`
  - `APP_URL=https://<domain>`
  - `DOMAIN=<domain>`
  - SMTP credentials for a real transactional email provider (e.g. Mailgun/Resend)
  - `BALE_BOT_TOKEN` / `TELEGRAM_BOT_TOKEN` / usernames

- [ ] **Deploy script** (`scripts/deploy.sh`)
  ```bash
  #!/bin/bash
  set -e
  cd /opt/voltana
  git pull origin main
  docker compose pull           # pull pre-built images if using a registry
  docker compose up -d --build api nginx
  docker compose run --rm migrate
  echo "Deploy complete $(date)"
  ```

- [ ] **SSL certificate setup** (documented in `docs/DEPLOY.md`)
  - `certbot --nginx -d <domain>` (initial issuance)
  - Cron renewal hook: `0 12 * * * certbot renew --quiet`

- [ ] **Docs: `docs/DEPLOY.md`** — step-by-step for a fresh VPS
  1. Clone repo + set `.env`
  2. Run `scripts/bootstrap-vps.sh`
  3. Issue TLS cert
  4. Copy systemd unit + `systemctl enable --now voltana`
  5. Run `scripts/deploy.sh` to verify green

## Acceptance Criteria

1. `scripts/bootstrap-vps.sh` runs idempotently on a fresh Ubuntu 22.04 LTS VPS
2. `https://<domain>/health` returns `{"status":"ok"}` with valid TLS (A rating on SSL Labs)
3. `http://<domain>` redirects to HTTPS (301)
4. UFW: only ports 22/80/443 open
5. `systemctl status voltana` shows `active (running)` after server reboot
6. `scripts/deploy.sh` deploys a new api version in < 60 s with zero manual steps
7. `/auth/` endpoints rate-limited at nginx level
8. Security headers present on all responses (`curl -I https://<domain>/health`)
9. `certbot renew --dry-run` exits 0
10. `docs/DEPLOY.md` accurate — a second operator can follow it cold

## Security Checklist

- [ ] SSH key-only auth (password auth disabled in `/etc/ssh/sshd_config`)
- [ ] UFW default deny-inbound applied before opening ports
- [ ] TLS 1.2+ only, no weak ciphers in nginx TLS config
- [ ] `APP_ENV=production` forces Secure+HttpOnly on refresh cookie
- [ ] `X-Real-IP` still the only trusted IP source (nginx sole ingress)
- [ ] No debug/dev endpoints exposed in production nginx config

## Notes

- **Domain**: operator must provision and point A record at VPS IP before certbot step.
- **SMTP**: production must use a real transactional provider (not MailHog). Resend or Mailgun recommended (free tier covers dev-scale volume).
- **VPS spec target**: Ubuntu 22.04 LTS, 2 vCPU / 4 GB RAM (same as dev host profile).
- The bot pollers (Bale/Telegram) should work from the VPS without the `extra_hosts` workaround — the VPS is in Iran and has direct access to Bale's API.
