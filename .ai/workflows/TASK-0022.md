# TASK-0022 — Advanced Efficiency Chart

**Phase**: 3 (feature)

**Reviewer**: dev_supervisor
**Persona**: pm (this spec) → feature (chart design + data shape) → developer (implements) → dev_supervisor
**Depends on**: TASK-0018 (✅ per-session kWh/100km) · TASK-0012 (✅ date-range filter + session detail)

## Goal
Show the user's driving efficiency (**kWh/100km**) as a **trend chart over time**, so they can see
improvement or degradation in real-world range. Built on the per-session efficiency values that
TASK-0018 introduced; no new backend endpoint is needed if the charging-session list already returns
`efficiency_kwh_per_100km` per session.

---

## Scope In

### Data
- Source: the `GET /v1/charging-sessions` list (already fetched in `pages/Charging.tsx`).
  Each session returned by the API includes `efficiency_kwh_per_100km` (nullable) since TASK-0018.
- **No new API endpoint**: the chart is built client-side from the session list, filtered to sessions
  where `efficiency_kwh_per_100km != null`.
- **Date-range filter**: reuse the existing `?from`/`?to` query params (TASK-0012) so the chart
  responds to the page-level filter.
- **Car filter**: if the user has multiple cars, allow filtering by car (existing multi-car filter
  from TASK-0012 applies).

### Chart (Recharts, already in `package.json`)
- **Primary line**: `efficiency_kwh_per_100km` over `started_at` date.
- **Average line**: a horizontal `ReferenceLine` at the mean efficiency of visible sessions.
- **Min/max range band**: a `<Area>` or `<ReferenceArea>` between the minimum and maximum values in
  the visible window (shows the spread at a glance).
- **Tooltip**: on hover, show date + efficiency value + car name.
- X-axis: dates (use `started_at`); Y-axis: kWh/100km.
- Chart should be `null`-safe: if fewer than 2 qualifying sessions exist, show a placeholder
  ("بارگذاری داده ناکافی — حداقل ۲ سفر با اودومتر لازم است").

### Placement
- **New card** on `pages/Index.tsx` (Dashboard), below the SOH battery chart — or as a new row.
- Feature persona to decide exact placement and whether to reuse `ChargingChart.tsx` structure or
  create a new `EfficiencyChart.tsx` component in `src/components/`.

## Scope Out
- Per-trip GPS efficiency (instantaneous, sub-session) — Phase 4 / OBD.
- Server-side chart data endpoint (client-side computation from the session list is sufficient).
- Exporting/downloading the chart.
- Outlier filtering (keep all data points; user can use date filter to exclude bad sessions).

## Key Design Decisions (for feature persona to resolve)
1. **Component location**: new `EfficiencyChart.tsx` in `src/components/` (preferred — reusable) or
   inline in `Index.tsx`?
2. **Chart height and breakpoints**: recommend `h-48` mobile, `h-64` desktop (same as `ChargingChart`).
3. **Average line color**: use `--primary` or a neutral `--muted-foreground` to contrast with the data line.
4. **Min/max band opacity**: suggest `opacity-20` fill so it doesn't obscure the primary line.
5. **Empty-state copy** (Persian): feature persona to draft.

## Acceptance Criteria
- [ ] Efficiency chart appears on the Dashboard with per-session kWh/100km plotted over time.
- [ ] Average reference line is visible and labeled.
- [ ] Min/max range band is rendered (semi-transparent fill between min and max).
- [ ] Chart updates when the date-range or car filter changes (uses the same TanStack Query key as

- [ ] Fewer than 2 qualifying sessions → placeholder text shown, no chart error.
- [ ] `npx tsc --noEmit` and `npm run build` pass.
